# 경유지·최종 방향·구간 반복 검증

대상 버전: `2026-07-13.41`

## 변경 내용

- `NavigateToPose` 순차 경로를 전용 콜백으로 분리했다.
- 경유지 성공 콜백 안에서 action server를 다시 기다리지 않고, 다음 목표를 별도 스레드에서 즉시 전송한다.
- 늦게 도착한 이전 경로 콜백은 세대 번호가 다르면 현재 목표와 runtime을 변경하지 않는다.
- 최종 목표를 지도에서 누른 뒤 드래그한 방향을 최종 yaw로 사용한다.
- 경유지와 최종 목표에 연속 번호를 부여하고, 시작 번호부터 종료 번호까지 반복할 수 있다.
- 마지막 지점 도착 후 `0.5~3600초` 휴식한 다음 선택 구간의 첫 지점으로 복귀해 자동 반복한다.
- 취소, 정지, 수동 운전, 새 목표, 서버 종료 시 반복 타이머와 오래된 콜백을 무효화한다.
- LiDAR A* 직접 추종은 반복 시 복귀 A* 경로를 안전하게 재생성할 수 없으므로 반복 모드와 동시 사용을 차단한다.

## 자동 검증

```bash
python -m py_compile server.py
node --check web/app.js
python -m unittest discover -s tests -v
```

검증 항목:

- 첫 번째 `NavigateToPose` 성공 후 두 번째 경유지 dispatch
- 오래된 result callback이 현재 goal handle을 지우지 않음
- 반복 휴식 후 cycle 증가 및 1번 지점 재전송
- 반복 번호 범위 pose/path 선택과 `routeIndex` 재번호 부여
- 한 지점만 지정한 잘못된 반복 구간 거부
- 기존 LiDAR, cmd_vel, SSH, 맵 설정 회귀 테스트

## 브라우저 프리뷰 검증

1. 경유지 2개와 최종 목표를 지정했다.
2. 최종 yaw를 `1.57 rad`로 지정하고 서버의 마지막 pose에 같은 값이 전달되는지 확인했다.
3. `#1 → #3`, 휴식 `0.5초`로 실행해 반복 회차가 계속 증가하는지 확인했다.
4. `#2 → #3`으로 범위를 바꾸면 서버가 pose 2개와 `sourceStart: 2`, `sourceEnd: 3`을 받는지 확인했다.
5. 반복 중 취소 후 추가 회차가 시작되지 않고 runtime이 `canceled`로 유지되는지 확인했다.

실제 TurtleBot에서는 Nav2 action server가 보이는 ROS2 Jazzy 서버에서 저속으로 첫 검증을 진행한다. 주행 상태의 지점 번호가 `1/N → 2/N → ...` 순서로 증가하고, 마지막 pose에서 지정한 yaw로 회전한 뒤 휴식 상태로 바뀌는지 확인한다.
